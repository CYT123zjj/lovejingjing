案例2:为虚拟机 server 配置Web访问控制
在 Web 网站 http://server0.example.com 的 DocumentRoot 目录下创建一个名为 private 的子目录，要求如下：
1）在server0.example.com的DocumentRoot目录下，private的子目录里书写网页文件index.html内容为  大圣偷偷归来
	[root@server0 ~]# mkdir /var/www/web1/private
	[root@server0 ~]# echo '大圣偷偷归来' > /var/www/web1/private/index.html
	[root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf
	<VirtualHost *:80>
	 ServerName server0.example.com
	 DocumentRoot /var/www/web1
	</VirtualHost>
	[root@server0 ~]# systemctl restart httpd
	[root@server0 ~]# firefox server0.example.com/private
	
2）此页面只能在本机浏览，但是从其他系统不能访问这个目录的内容
	[root@server0 ~]# vim /etc/httpd/conf.d/nsd02.conf 
	<Directory "/var/www/web1/private">
	 Require ip 172.25.0.11
	</Directory>

	测试
	[root@desktop0 ~]# firefox server0.example.com/private


案例3:为虚拟机 server 使用自定Web根目录(保证SELinux为开启状态)

调整 Web 站点 http://server0.example.com 的网页目录，要求如下：
1）新建目录 /webroot，作为此站点新的网页目录
	 mkdir /webroot
	 [root@server0 ~]# echo '<h1> wo shi webroot' > /webroot/index.html
	 [root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf 
	<VirtualHost *:80>
	 ServerName server0.example.com
	 DocumentRoot /webroot
	</VirtualHost>
	 [root@server0 ~]# vim /etc/httpd/conf.d/nsd02.conf 
	<Directory "/webroot">
	 Require all granted
	</Directory>
	
2）确保站点 http://server0.example.com  仍然可访问
	[root@server0 ~]# getenforce 
	Enforcing
	[root@server0 ~]# ls -Zd /var/www
	[root@server0 ~]# ls -Zd /webroot/
	[root@server0 ~]# chcon -R --reference /var/www /webroot
	[root@server0 ~]# systemctl restart httpd
	[root@server0 ~]# firefox server0.example.com


案例4：为虚拟机 server 部署动态WSGI站点
为站点 webapp0.example.com 配置提供动态Web内容，要求如下：
1）此虚拟主机侦听在端口8909
	[root@server0 conf.d]# vim /etc/httpd/conf.d/nsd01.conf 
	Listen 8909
	<VirtualHost *:8909>
	 ServerName webapp0.example.com
	 DocumentRoot /var/www/web3
	</VirtualHost>

2）测试网页从以下地址下载，不要作任何更改http://classroom/pub/materials/webinfo.wsgi
	[root@server0 conf.d]# cd /var/www/web3
	[root@server0 web3]# wget http://classroom.example.com/pub/materials/webinfo.wsgi
	[root@server0 conf.d]# yum -y install mod_wsgi
	[root@server0 conf.d]# vim /etc/httpd/conf.d/nsd01.conf 
	Listen 8909
	<VirtualHost *:8909>
	 ServerName webapp0.example.com
	 DocumentRoot /var/www/web3
	 WsgiScriptAlias / /var/www/web3/webinfo.wsgi
	</VirtualHost>
	[root@server0 conf.d]# systemctl restart httpd
 
3）从浏览器访问 http://webapp0.example.com:8909 可接收到动态生成的 Web 页面
	[root@server0 conf.d]# semanage port -l | grep http
	[root@server0 conf.d]# semanage port -a -t http_port_t -p tcp 8909
	[root@server0 conf.d]# systemctl restart httpd
	[root@server0 conf.d]# firefox webapp0.example.com


案例5：配置安全Web服务
为站点 https://www0.example.com 配置TLS加密 
1）一个已签名证书从以下地址获取 http://classroom/pub/tls/certs/server0.crt 
	[root@server0 web3]# cd /etc/httpd/conf.d
	[root@server0 web3]# yum -y install mod_ssl
	[root@server0 conf.d]# cd /etc/pki/tls/certs/
	[root@server0 certs]# wget http://classroom.example.com/pub/tls/certs/server0.crt
	[root@server0 certs]# vim /etc/httpd/conf.d/ssl.conf 
	DocumentRoot "/var/www/web2"
	ServerName www0.example.com:443
	SLCertificateFile /etc/pki/tls/certs/server0.crt


	

2）此证书的密钥从以下地址获取 http://classroom/pub/tls/private/server0.key 
	[root@server0 tls]# cd private/
	[root@server0 private]# wget http://classroom.example.com/pub/tls/private/server0.key
	[root@server0 certs]# vim /etc/httpd/conf.d/ssl.conf
	SSLCertificateKeyFile /etc/pki/tls/private/server0.key


3）根证书的签名授权信息从以下地址获取 http://classroom/pub/example-ca.crt
	[root@server0 certs]# wget http://classroom.example.com/pub/example-ca.crt
	[root@server0 certs]# vim /etc/httpd/conf.d/ssl.conf
	SSLCACertificateFile /etc/pki/tls/certs/example-ca.crt

	测试
	[root@server0 certs]# firefox https://www0.example.com
